Motivation

Thinking about huge design spaces can be quite daunting. When designing a new robot for some task, you might think about the millions of different choices you can make. Wheels or legs? Should the mouth move or should the sound just play through motionless speakers near the mouth? Should the robot blink?

Often, many of these low level choices will be overlooked, and what really matters is the impression that your design gives. Low-level choices are easier to make if you have formulated an end-goal that you are trying to design toward.

A Dataset to Describe the Design Space

In order to describe the design space, we take a data-driven approach. A dataset of 165 Socially Interactive Robots was developed to understand how people expect the robot to interact both socially and functionally. We crowd-sourced ratings from workers on Mechanical Turk to find average expectations for each of the robots in the dataset.

Try exploring the dataset below!

  • The scatter plot places similar robots near eachother; Hover over each datapoint to see the robot it represents.
  • Press the buttons to update the colors the graph uses to quickly see the values for each social or functional expectation.
  • We assembled word clouds of each robot to understand what words people commonly used to describe each robot.

    • Study Information

    The following 3 files contain the cleaned information collected from the studies. For all studies, Likert questions are measured on a scale of -3 to 3 (where -3 is strongly disagree, and 3 is strongly agree with intermediate values as shown in the videos of the question pages), and semantic-differential scale questions are measured on a scale of 1 to 9 (where 1 is the left pole, and 9 is the right pole, as shown in the video). All demographic information is assigned to a key described seperately to help reduce personal researcher biases toward demographic groups when exploring the data.

    Note: All the CSVs are tab-delimted

    • CleanedMetaphorResponses(Study1).csv contains the cleaned information collected from the design metaphor study.
    • CleanedSocialResponses(Study2).csv contains the cleaned information collected from the social perception study.
    • CleanedFunctionalResponses(Study3).csv contains the cleaned information collected from the functional perception study.

    Auxillary Study Information

    • DemographicInformationMapping.csv contains the human-readable items that represent the items in the previous 3 files.
    • RobotAverages.csv contains the information from study2 and study3, aggregated by robot. All values represent means of all user-reported metrics for the given robot.

    Robot Database Information

    • Robot Descriptors - Included Robots.csv gives a table of all of the robots included in the study, as well as their coded features. These features were used to make the t-SNE plot to visually show the relationships between different robots.
    • Robot Descriptors - Feature Explanation.csv gives a human-understandable description of what the features listed in the database mean.
    • MetaphorClassifications.csv contains the classifications that were assigned to each unique metaphor.
    • Robot Images.pdf contains the stimuli provided during the study.
    • MetaphorCounts.csv contains the top 3 metaphors for each robot and their counts.
